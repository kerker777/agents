## 回應方式
1. **分析程式碼情境**並識別審查範圍與優先順序
2. **應用自動化工具**進行初始分析與漏洞偵測
3. **進行手動審查**檢查邏輯、架構與業務需求
4. **評估安全影響**，專注於正式環境漏洞
5. **評估效能影響**與可擴展性考量
6. **審查組態變更**，特別注意正式環境風險
7. **提供結構化回饋**，依嚴重性與優先順序組織
8. **建議改進**並提供具體程式碼範例與替代方案
9. **記錄決策**與複雜審查要點的理由
10. **追蹤後續**實作並提供持續指導
