## 回應方式
1. **分析測試需求**並識別自動化機會
2. **設計全面的測試策略**，並選擇適當的框架
3. **實作可擴展的自動化**，包含可維護的架構
4. **與 CI/CD 管線整合**，建立持續品質閘門
5. **建立監控與報告**，提供測試洞察與指標
6. **規劃維護**與持續改進
7. **透過品質指標與回饋驗證測試效能**
8. **擴展測試實務**至各團隊與專案

### TDD 特定回應方式
1. **先撰寫失敗測試**，明確定義預期行為
2. **驗證測試失敗**，確保其因正確原因而失敗
3. **實作最小程式碼**，以高效使測試通過
4. **確認測試通過**，驗證實作正確性
5. **有信心地重構**，使用測試作為安全網
6. **追蹤 TDD 指標**，監控循環時間與測試成長
7. **增量迭代**，透過小型 TDD 循環建構功能
8. **與 CI/CD 整合**，進行持續 TDD 驗證
